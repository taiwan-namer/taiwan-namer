# Spec: taiwannannamer-conversion-v1

## Background
Taiwannamer.com 是一個提供網域名稱註冊與架站服務資訊的網站，主要營收來源為聯盟行銷（Affiliate Marketing）。目前網站流量未能有效轉化為聯盟連結的點擊與最終成交（Domain 註冊 / 架站服務購買），導致潛在收益流失。本專案目標為透過優化使用者體驗、資訊架構與引導設計，顯著提升網站的轉換率，進而增加聯盟收入。

## Requirements
**R1: 首頁價值主張與行動呼籲優化**
*   必須清晰傳達網站的核心價值（例如：尋找最優惠的網域註冊與架站方案）。
*   必須在首屏（Above the Fold）放置明確、吸引人的主要行動呼籲按鈕。
*   必須減少首頁的資訊雜訊，聚焦於引導使用者進入「服務比較」或「推薦」頁面。

**R2: 服務比較/推薦頁面重設計**
*   必須建立獨立、內容清晰的「網域註冊商比較」與「架站服務比較」頁面。
*   必須以表格或卡片形式，直觀地呈現不同服務商（如 GoDaddy, Namecheap, Shopify, Wix）的關鍵資訊（如價格、特色、優缺點）。
*   必須在每個服務選項旁，放置醒目且可追蹤的聯盟連結按鈕（例如：「前往官網查看優惠」）。
*   必須提供簡單的篩選或排序功能（例如：依價格、依推薦度）。

**R3: 內容引導與情境式推薦**
*   必須在相關的部落格文章或指南內容中，自然地嵌入情境式推薦與聯盟連結。
*   例如：在「如何選擇網域名稱」文章結尾，推薦1-2家當前有優惠的註冊商並提供連結。
*   必須確保內容中的推薦感覺真誠、有幫助，而非強迫推銷。

**R4: 信任信號與社會證明強化**
*   必須在關鍵頁面（如首頁、比較頁）展示信任元素，例如：網站資歷、幫助過的用戶數、合作品牌標誌（若有可能）。
*   必須考慮加入使用者評價、評分或案例分享區塊。
*   必須確保網站設計呈現專業、可信賴的外觀。

**R5: 聯盟連結追蹤與分析**
*   必須能追蹤每個聯盟連結的點擊次數（Click-through Rate, CTR）。
*   必須確保網站結構利於後續分析使用者在轉換漏斗中的行為（例如：從首頁 > 比較頁 > 點擊連結的流程）。
*   必須提供清晰的數據看板或報告方式，讓管理者了解各頁面與連結的表現。

**R6: 行動裝置體驗優化**
*   所有優化必須在行動裝置上提供同樣流暢、易用的體驗。
*   行動裝置上的行動呼籲按鈕必須易於點擊，表格或比較資訊必須易於閱讀。

## User Stories
**US1: 尋找便宜網域的新手用戶**
*   作為一名想建立個人部落格但預算有限的新手，
*   我希望一進入網站就能明白這裡可以幫我找到高性價比的網域註冊商，
*   這樣我才能快速產生信任並繼續瀏覽，而不會因困惑而離開。

**US2: 想比較架站方案的小企業主**
*   作為一名想為公司建立官網的小企業主，
*   我希望有一個頁面能清晰比較 Shopify、Wix 等平台的費用和功能，
*   這樣我才能輕鬆做出決定，並透過網站提供的連結前往購買，省去自行搜尋比較的時間。

**US3: 透過文章學習的潛在客戶**
*   作為一名正在閱讀「最佳網域後綴推薦」文章的讀者，
*   我希望在文章結尾能看到筆者根據內容推薦的註冊商及當下優惠連結，
*   這樣我可以在獲得資訊後立即行動，把握可能限時的優惠。

**US4: 網站管理者（內部）**
*   作為網站的管理者，
*   我希望能夠一目了然地看到哪個服務商比較頁面或哪篇部落格文章帶來的點擊最多，
*   這樣我才能知道該優化哪些內容或加強推廣哪些聯盟合作。

## Acceptance Criteria
1.  專案上線後一個月內，整體聯盟連結點擊率（CTR）相較於優化前基準提升 **20%**。
2.  經由使用者測試（至少5人），所有測試者能在 **15秒內** 於首頁理解網站用途，並找到前往服務比較頁面的入口。
3.  「網域註冊商比較」與「架站服務比較」頁面的跳出率（Bounce Rate）低於網站平均跳出率的 **50%**。
4.  在行動裝置上，所有行動呼籲按鈕的點擊區域大小符合無障礙標準（至少44x44像素），且頁面載入速度在 4G 環境下於 **3秒內** 完成可互動。
5.  後台能夠產出報表，區分不同頁面來源（首頁、比較頁、文章頁）的連結點擊數據，誤差率低於 **5%**。

## Out of Scope
1.  開發全新的後台管理系統。分析數據將優先利用現有或第三方工具（如 Google Analytics, Bitly）進行追蹤與呈現。
2.  大幅改寫現有部落格文章內容。本階段僅針對文章內的「引導區塊」與連結嵌入進行優化。
3.  建立使用者帳戶系統、收藏夾或個人化推薦引擎。
4.  直接處理金流或與聯盟平台（如 ShareASale, Commission Junction）的 API 深度串接。連結追蹤將以添加 UTM 參數或使用連結縮短服務為主。
5.  搜索引擎優化（SEO）的全面調整，儘管部分優化可能對 SEO 有間接幫助，但這不是本階段的主要目標。
