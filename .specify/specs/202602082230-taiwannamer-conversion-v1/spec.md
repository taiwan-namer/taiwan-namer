# Spec: taiwannamer-conversion-v1

## Background
現有網站 taiwannamer.com 提供中文網域名稱生成服務，但轉換率（用戶點擊聯盟連結前往註冊網域或購買架站服務）有待提升。為優化商業成效，本專案旨在重新設計使用者流程與介面，強化「產生名稱」到「採取行動」之間的引導，並建立可追蹤的數據框架，以有效提升聯盟行銷收入。初版目標專注於提升外部連結點擊率至 3% 以上。

## Requirements
**R1. 結果引導流程**
在生成名稱結果頁面，必須提供清晰、逐步的行動呼籲（Call-to-Action, CTA）引導，引導用戶完成從選擇名稱到購買的旅程。引導內容需包含：
1.  提示用戶從列表中選擇一個喜歡的網域名稱。
2.  提供按鈕，讓用戶能「立即查詢」該網域的可用性與註冊價格（優先顯示 .com 及 .tw 網域）。
3.  在查詢按鈕旁或結果附近，顯示緊迫性文案（例如：「建議立即購買，避免被搶註」）。
4.  提供另一個明顯的按鈕或區塊，引導有架站需求的用戶前往合作夥伴（如 Bluehost）的服務頁面。

**R2. 結果卡片互動功能**
每個生成出的網域名稱結果卡片，必須提供以下直接互動功能：
1.  **一鍵複製**：點擊後將該網域名稱（純文字，如 `taiwannamer.com`）複製到用戶的剪貼簿。
2.  **收藏**：點擊後將該網域名稱收藏至本地（使用瀏覽器的 localStorage），並在介面上提供查看已收藏名稱的入口。
3.  **分享**：點擊後生成一個包含當前頁面狀態（或特定名稱）的分享連結，並將此連結複製到用戶剪貼簿，供用戶分享。

**R3. 生成前偏好篩選**
在用戶觸發名稱生成前，應提供簡單的選項讓用戶設定偏好，使生成結果更貼近其需求。選項需包含：
1.  **用途**：可選「品牌 / 公司 / 店名 / 個人IG」等。
2.  **語言**：可選「中文 / 英文 / 中英混合」。
3.  **風格**：可選「專業 / 可愛 / 潮流 / 霸氣」等。
用戶選擇的偏好應顯著影響生成演算法的結果排序或內容。

**R4. 聯盟連結追蹤路由**
所有指向外部合作夥伴（如 Namecheap, GoDaddy, Bluehost）的聯盟連結，必須改為透過站內路由進行跳轉。
1.  路由格式統一為：`/go/[vendor]?domain=xxx`
    *   例如：`/go/namecheap?domain=taiwannamer`
    *   例如：`/go/bluehost` (無需 domain 參數時)
2.  此路由頁面需負責記錄點擊事件，並立即（或短暫延遲後）將用戶重新導向至真實的外部聯盟連結。

**R5. 關鍵事件追蹤**
必須實作追蹤機制，至少記錄以下關鍵使用者事件：
1.  `generate_click`：用戶點擊「生成名稱」按鈕。
2.  `outbound_click_[vendor]`：用戶點擊前往特定廠商（如 namecheap, godaddy, bluehost）的連結。
3.  `copy_domain`：用戶複製某個網域名稱。
4.  `favorite_add`：用戶收藏某個網域名稱。
追蹤數據需可被查閱，最小可行產品（MVP）階段允許使用簡單的後台日誌或儀表板呈現，但必須能清晰看到各事件的發生次數。

## User Stories
**US1. 作為尋找店名的創業者**
我希望在輸入關鍵字後，能先選擇「店名」用途和「可愛」風格，
以便系統生成的名稱更符合我的咖啡廳或小店的形象。

**US2. 作為已找到心儀域名的用戶**
當我看到一個喜歡的域名（如 `lovelycafe.tw`）時，
我希望可以一鍵複製它，並立刻查詢它的註冊價格與可用性，
以便我能快速進行購買，不用手動輸入或切換視窗。

**US3. 作為猶豫不決的用戶**
當我看到多個喜歡的域名但無法立刻決定時，
我希望可以將它們加入收藏清單暫存，
以便我稍後可以回來比較並做出最終決定。

**US4. 作為需要一站式服務的用戶**
當我確定要註冊某個域名並想立刻架設網站時，
我希望在查詢域名後，能被清楚地引導至架站服務（如 Bluehost），
以便我能無縫開始架站流程。

**US5. 作為網站管理者**
我希望能看到用戶點擊「生成」、「查詢域名」、「前往架站」等按鈕的次數，
以便我了解哪個環節的轉換效果最好，並作為未來優化的依據。

## Acceptance Criteria
1.  當用戶完成名稱生成後，頁面顯眼處必須出現符合 **R1** 描述的引導文字與按鈕。
2.  每個域名結果旁，必須有符合 **R2** 描述的「複製」、「收藏」、「分享」功能圖示或按鈕，且功能運作正常（複製成功需有視覺回饋）。
3.  在主要的生成輸入區旁或上方，必須有符合 **R3** 描述的「用途」、「語言」、「風格」下拉選單或選項卡，且選擇不同選項後生成的結果應有可感知的差異。
4.  點擊任何「查詢價格」或「前往架站」按鈕，瀏覽器網址列必須先變更為符合 **R4** 格式的站內路由（如 `/go/namecheap?domain=xxx`），隨後才跳轉至外部網站。
5.  後台或指定的日誌系統中，必須能查詢到 **R5** 所列舉的每一種事件類型及其觸發次數。
6.  所有新功能不應導致網站主要功能（名稱生成）的效能顯著下降或出現錯誤。
7.  收藏功能需基於 localStorage 實作，用戶在相同瀏覽器內再次訪問網站時，應能看到之前收藏的域名清單。

## Out of Scope
1.  實作用戶帳號系統、登入/註冊功能。
2.  建立付費牆或高級功能訂閱制。
3.  對網站進行全面的搜尋引擎優化（SEO）架構改版。
4.  開發極度複雜或智慧化的命名演算法；本次優化以提升現有流程的轉換率為首要目標。
5.  設計複雜的數據分析儀表板；MVP 階段僅需確保數據能被記錄與基本查閱。
6.  針對行動裝置或不同瀏覽器進行全面的響應式設計重構（除非嚴重影響新功能使用）。
7.  實作第三方分享（如 Facebook, Twitter）按鈕；分享功能僅限複製連結。
