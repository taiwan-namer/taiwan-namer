# Spec: 台灣命名測試 UI

## Background
「台味命名大師」是一個以台灣在地文化為核心的 AI 網域命名工具，旨在協助使用者透過諧音梗、在地文化意涵與算命筆畫等元素，產生具備台味特色的創意網域名稱。產品目標是提供從「命名靈感」到「網域查價」、「註冊購買」乃至「網站架設」的一條龍服務體驗，並透過聯盟行銷機制與主流網域註冊商及主機商合作，創造商業價值。

現有流程已實現核心功能，但使用者體驗、數據管理與商業擴展性仍有優化空間。本規格書旨在定義產品需求，並提出結構化優化方向，以提升產品穩定性、使用者參與度與轉換率。

## Requirements

### R1: 核心命名體驗
必須提供流暢、直覺的 AI 網域命名生成體驗。
*   **R1.1**: 使用者應能在首頁輸入一個或多個關鍵字（例如：珍珠奶茶、好運），以獲得個性化的網域建議。
*   **R1.2**: 若使用者未輸入關鍵字，系統應使用預設關鍵字（如：珍珠奶茶、好運）產生範例結果，以展示產品能力。
*   **R1.3**: AI 生成的每個網域建議，必須清晰呈現以下資訊：
    *   網域名稱（含後綴，如 `.com`, `.tw`）
    *   對應的中文名稱
    *   名稱的寓意或解釋
    *   該頂級域（TLD）的參考起價（以新台幣 NT$ 顯示）
*   **R1.4**: 生成的網域列表應根據預設規則（例如：優先顯示 `.com`、`.tw`）進行排序。

### R2: 網域操作與互動
使用者必須能對感興趣的網域進行後續操作。
*   **R2.1**: 提供「一鍵複製」網域名稱至剪貼簿的功能。
*   **R2.2**: 提供「收藏」功能，將網域儲存於使用者的瀏覽器本地儲存（localStorage）中，並可於特定頁面檢視已收藏清單。
*   **R2.3**: 提供「分享」功能，生成包含該網域參數的專屬連結，方便使用者傳播。
*   **R2.4**: 針對每個網域建議，提供明確的按鈕，引導使用者至合作註冊商（GoDaddy, Namecheap）進行即時查價與購買。

### R3: 合作商導流與追蹤
必須建立可靠且可追蹤的導流機制至合作夥伴。
*   **R3.1**: 需建立站內跳轉端點（如 `/go/[vendor]`），用於記錄點擊並安全地轉導至外部合作商網站。
*   **R3.2**: 跳轉時必須帶入必要的聯盟行銷參數，以確保行銷成效可被正確歸因。
*   **R3.3**: 每次跳轉必須記錄詳細的點擊日誌，至少包含：時間戳記、來源頁面、網域名稱、合作廠商、頂級域。
*   **R3.4**: 需根據網域後綴智慧判斷可用的合作註冊商（例如：`.tw` 網域不應顯示不支援的 Namecheap 選項）。

### R4: 內容與教育推廣
需透過內容建立產品專業度並推廣相關服務。
*   **R4.1**: 首頁需包含「架站主機推廣區塊」，明確引導已完成命名的使用者進行下一步，並導向合作主機商（如 Bluehost）。
*   **R4.2**: 需建立「部落格」系統，發布與網域註冊、網站架設、平台教學相關的文章，以提供附加價值並提升 SEO。
*   **R4.3**: 需提供完整的靜態頁面，包括「關於我們」、「聯絡我們」、「隱私權政策」與「服務條款」，以建立信任感與合規性。

### R5: 數據分析與管理
必須具備基礎的數據洞察能力以優化營運。
*   **R5.1**: 需在前端關鍵使用者互動點（如：生成命名、收藏、複製、點擊外部連結）埋設分析事件。
*   **R5.2**: 需提供簡單的後台介面，讓管理員能夠檢視從 `/go/[vendor]` 跳轉所累積的點擊統計數據，例如總點擊數、依廠商分類的數量、最近點擊記錄等。

## User Stories

### US1: 尋找創業點子的使用者
身為一位想在台灣開設特色手搖飲店的創業者，
我希望輸入「奶茶、創意」等關鍵字來獲得有趣的網域名稱，
以便為我的新品牌找到一個好記、有台味且未被註冊的官方網址。

### US2: 節省比價時間的自由工作者
身為一位經常需要為客戶購買網域的自由網頁設計師，
當我從 AI 結果中找到喜歡的網域時，
我希望可以直接點擊按鈕前往 GoDaddy 或 Namecheap 查詢即時價格與購買，
這樣我就不需要手動在不同註冊商網站間反覆查詢比價。

### US3: 需要架站指引的新手站長
身為一個剛買好網域、對架設網站毫無頭緒的新手，
當我在首頁下方看到 Bluehost 主機方案與架站教學文章時，
我希望可以點擊進入詳細的圖文教學，
以便我能一步步跟著操作，順利將我的網域架設成可訪問的網站。

### US4: 監控行銷成效的管理員
身為產品的管理員，
我希望登入後台後能一目了然地看到各合作廠商（GoDaddy, Namecheap, Bluehost）帶來的點擊流量，
以便我評估聯盟行銷管道的成效，並作為未來商業合作的參考依據。

## Acceptance Criteria

### 針對首頁命名流程 (R1, R2)
1.  **AC1.1**: 在首頁輸入框鍵入「芒果冰」並按下「開始算命」按鈕後，頁面應在 5 秒內顯示出約 6 個包含「芒果冰」諧音或寓意的網域建議卡片。
2.  **AC1.2**: 每個網域卡片上必須顯示：網域（如 `MangoIce.tw`）、中文名（如「芒好了」）、寓意（如「吃了芒果冰，一切『芒』好了」）、以及「NT$ XXX 起」的價格標示。
3.  **AC1.3**: 點擊卡片上的「複製」圖示，瀏覽器應提示「已複製網址」，且剪貼簿內容應為該網域名稱。
4.  **AC1.4**: 點擊卡片上的「愛心」圖示，圖示應變為實心，且該網域應被加入「我的收藏」清單中。

### 針對合作商導流 (R3)
1.  **AC2.1**: 點擊 `.com` 網域卡片上的「GoDaddy 查價」按鈕，應先導向站內 `/go/godaddy?domain=xxx.com`，隨後立即（302 跳轉）導向 GoDaddy 官網的該網域搜尋頁面，且網址應包含聯盟追蹤參數。
2.  **AC2.2**: 點擊 `.tw` 網域卡片時，不應出現「Namecheap 查價」的按鈕選項。
3.  **AC2.3**: 完成 AC2.1 的跳轉後，後台點擊統計中應新增一筆包含正確 domain、vendor (`godaddy`)、tld (`com`) 及時間的記錄。

### 針對數據管理 (R5)
1.  **AC3.1**: 管理員訪問 `/admin/clicks` 頁面時，應能看到一個表格或儀表板，顯示總點擊次數、以及分別流向 GoDaddy、Namecheap、Bluehost 的次數。
2.  **AC3.2**: 該頁面應能列出最近 50 筆點擊的詳細記錄，包含點擊時間與目標網域。

## Out of Scope
以下為本階段明確**不包含**的需求：
1.  使用者帳號系統（註冊、登入、個人檔案）。
2.  跨裝置同步收藏清單（目前僅依賴 localStorage）。
3.  複雜的網域後綴過濾器或自定義排序功能（例如讓使用者自選偏好 `.io` 優先）。
4.  即時網域可用性檢查（目前僅顯示參考價格，不進行實際 whois 查詢）。
5.  完整的後台管理系統（如使用者管理、內容發布系統 CMS），僅包含基礎點擊數據檢視。
6.  行動裝置應用程式（App）。
7.  多國語言（Language Localization）支援。
8.  與非指定合作商（如 Gandi, Cloudflare）的整合。
9.  線上金流或直接在站內完成網域購買交易。

## Clarify 決策補充（MVP）

- 多個關鍵字合併作為單次生成提示，不拆分流程。
- 網域排序固定為：.com → .tw → .io → .ai，其餘忽略。
- 收藏結果顯示於結果頁下方彈出區塊，不設獨立頁。
- 所有國際後綴導向 Namecheap，.tw 導向台灣註冊商。
- 架站推薦區塊固定顯示 Bluehost 單一 CTA。
- 部落格僅為靜態文章頁，不設 CMS。
- 後台為簡單密碼保護頁面讀取事件 JSON log。
- 網域價格顯示為靜態參考值，不串 API。
- 聯盟連結使用固定 affiliate 參數。
- 分享連結包含生成參數並可還原結果頁。
- 預設首頁顯示 5 組範例結果。

## Clarify 決策補充（MVP 定義）

### 關鍵字處理
- 多個關鍵字合併為單次 AI 生成提示，不拆分搜尋流程。

### 網域價格顯示
- 所有 TLD 顯示靜態參考價（人工設定區間值，例如 .com NT$300、.tw NT$800）。
- 價格僅作為引導用途，不保證即時準確。

### 收藏功能呈現
- 收藏清單顯示於結果頁下方彈出區塊，不設獨立頁面。

### 註冊商導向規則
- 所有國際後綴（.com, .io, .ai）導向 Namecheap
- .tw 導向單一台灣註冊商（先固定一家）

### 架站推廣區塊
- 結果頁底部固定顯示 Bluehost 推薦區塊，含單一 CTA 按鈕
- 點擊需記錄事件後跳轉

### 部落格系統範圍
- 僅靜態文章頁（寫在專案內）
- 不提供後台發文與 CMS 功能

### 數據與後台
- 所有事件寫入 server-side JSON log
- 後台為受密碼保護頁面，顯示：
  - 各事件總次數
  - 各聯盟點擊次數
- 不提供即時分析與進階篩選

### 驗收標準補充
- 首頁未輸入關鍵字時自動顯示 5 組範例結果
- 網域列表必須依序排序：
  .com → .tw → .io → .ai
- 每個 CTA 點擊必須寫入事件 log
- 分享連結需可還原同組生成結果頁

## 驗收標準補充（內容與後台）

### R4 內容區塊
- 架站主機推廣區塊需在結果頁固定可見
- 點擊 CTA 必須觸發 outbound_click_bluehost 事件後跳轉
- 部落格至少提供 3 篇靜態文章頁可正常訪問
- 靜態頁面至少包含：關於我們、聯絡方式、隱私權政策

### R5 後台總覽呈現
- 後台首頁以摘要卡片顯示：
  - 總生成次數
  - 各聯盟商點擊總數
- 下方表格顯示最近 50 筆事件紀錄
